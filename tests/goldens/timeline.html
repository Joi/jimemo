Building jimemo

From a one-page spec to five seed templates, six weeks end to end.

jimemo turns a markdown or YAML content file into a single polished HTML page — no server, no build step, nothing fetched at view time. This is the project's own history, kept as a jimemo timeline.

  1. 2026-05-20

    Design spec drafted

    First pass at the shape of the tool: a manifest describes a template's slots, content fills them, and the renderer inlines everything — CSS, images, fonts — into one file.

  2. 2026-06-01

    Vendoring decided over pip install

    Runtime dependencies (Jinja2, MarkupSafe, python-Markdown, PyYAML) would be vendored as pinned, checksummed pure-Python copies, so ./jimemo runs from a clone with nothing to install.

  3. 2026-06-09

    Repo scaffold and CI merged

    --version, doctor, and template discovery landed alongside a GitHub Actions workflow pinned to full commit SHAs.

    milestone

  4. 2026-06-20

    Toolkit design system lands

    Tokens, base typography, and nine components — page-header, stat-tile, card-grid, timeline, data-table, figure-block, badge, toc, tree — each with a Jinja macro and light, dark, and print support.

  5. 2026-06-28

    Render pipeline ships

    jimemo render assembles a template and its content into one HTML file: CSS inlined, local images converted to data URIs, lint fails closed on any external script.

  6. 2026-07-01

    Suggest and render auto

    A deterministic scorer, no LLM in the loop, reads a content file's shape and ranks which template fits best, so render auto can pick one on its own.

  7. 2026-07-04

    Security hardening pass

    Markdown output now runs through an HTML sanitizer before it reaches the page, and the lint step gained checks for inline event handlers and script-scheme URLs.

  8. 2026-07-05

    Five seed templates ship

    Briefing, photo-catalog, timeline, genealogy, and data-dashboard land together, each with a golden test and a real sample — ready to dogfood on an actual write-up.
